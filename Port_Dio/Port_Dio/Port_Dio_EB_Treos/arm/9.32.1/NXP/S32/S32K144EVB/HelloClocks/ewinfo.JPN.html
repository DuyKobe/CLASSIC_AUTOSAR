Example description

  
HelloClocks 
    
DESCRIPTION
===========
    This example project shows how to use the IAR Embedded Workbench for ARM
  to develop code for NXP S32K144EVB board based on NXP Application Note AN5413.
  This project provides common initialization for clocks and an LPIT channel
  counter function. Core clock is set to 80 MHz. LPIT0 channel 0 is configured
  to count one second of SPLL clocks. Software polls the channelâ€™s timeout flag
  and toggles the GPIO output to the LED when the flag sets.
  
   Note: The default flash loader will prevent writing at addresses
  0x408-0x40F and will program the FSEC and FOPT bytes with 0xFE and 0x7D
  respectively. This will keep the microcontroller unsecured and will allow it
  to boot from the flash memory. If you need to program the FSEC, FOPT, FEPROT 
  and FDPROT you must use the flash loader with --enable_config_write parameter

COMPATIBILITY
=============
  The example project is compatible with NXP S32K144EVB.
  By default, the project is configured to use the I-Jet SWD interface.
  It can also use NXP onboard OpenSDA (PE Micro).

CONFIGURATION
=============
  The Workspace contains HelloClocks example with few configurations.

  Flash Debug: The example is flashed into the embedded flash memory of 
			   the microcontroller.
  RAM Debug:   The example is loaded into the internal RAM of the microcontroller.
  
GETTING STARTED
===============

  1) Start the IAR Embedded Workbench for ARM.

  2) Connect the I-Jet

  3) Power on the board.

  4) Select ADC RAM or Flash Configuration

  5) Press Ctrl+D or use the Download and Debug button to start a debug session
